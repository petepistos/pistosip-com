What makes Skopein different

The difference between asking and inspecting.

API-based platforms pull self-reported data from vendor portals. They confirm your tools exist. Skopein reads what those tools are actually doing — the configuration state of your environment as it exists right now, not as a dashboard represents it.

API-based platforms

Connects to vendor APIs and pulls self-reported configuration data

Confirms a firewall exists — not whether its rules are defensible

Cannot see on-premises Windows environments, GPO, or local policy state

Cannot assess external attack surface — DNS, SSL/TLS, email authentication

Marks the checkbox. Misses the misconfiguration.

Skopein

Interrogates the actual environment — reads what is configured, not what is reported

Reads the firewall ruleset — flags the broad inbound rule exposing RDP

Runs locally on Windows endpoints and domain controllers — reads GPO, AD, local policy

Scans the external attack surface — DNS, SSL/TLS, SPF/DMARC/DKIM, security headers

Finds what has never been checked. Surfaces what an attacker finds first.

What Skopein actually finds

The things no vendor portal will tell you.

Real findings from Skopein assessments — in environments that had passed internal reviews, because those reviews asked the tools, and the tools said everything was fine.

NETWORK PERIMETER

  • Broad inbound rule allowing all traffic to internal subnets — present since 2019
  • Botnet filter license expired — signatures not updating for 14 months
  • Group VPN using deprecated encryption — DES/3DES in active use
  • SSL VPN certificate self-signed — client warnings suppressed by policy

ACTIVE DIRECTORY

  • 23 accounts inactive for 90+ days — still enabled and licensed
  • Service account with Domain Admin — password set 6 years ago, never rotated
  • Password minimum: 8 characters — no MFA for legacy authentication
  • Five users with direct Domain Admin — no privileged access workstation

EXTERNAL POSTURE

  • DMARC set to p=none — domain spoofing not prevented
  • SSL certificate expires in 11 days — no automated renewal configured
  • TLS 1.0 still negotiated on client portal — PCI DSS 4.0 non-compliant
  • Security headers missing: X-Frame-Options, CSP, HSTS

ENDPOINT POLICY

  • BitLocker not enforced on 8 of 23 endpoints — GPO not applied to OU
  • Local admin enabled on all workstations — shared password
  • Windows Firewall disabled on three servers — never restored after troubleshooting
  • EDR agent absent on two critical servers — excluded during upgrade
