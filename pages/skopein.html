What makes Skopein different

The difference between asking and inspecting.

Evidence collectors pull API data to populate checkboxes. They tell you what your tools report about themselves. Skopein reads what is actually there — the state of your environment as it exists right now, not as a vendor dashboard represents it.

API-based evidence collectors

Connects to vendor APIs and pulls self-reported configuration data

Confirms a firewall exists — not whether its rules are sound

Cannot see on-premises Windows environments, domain controllers, or local policy state

Cannot assess external attack surface — DNS, SSL/TLS, email authentication posture

Marks the checkbox. Misses the misconfiguration.

Skopein

Interrogates the actual environment — reads what is configured, not what is reported

Reads the firewall ruleset — flags the broad inbound rule that exposes RDP

Runs locally on Windows endpoints and domain controllers — reads GPO, AD, local policy state

Scans the external attack surface — DNS, SSL/TLS, SPF/DMARC/DKIM, security headers

Finds what has never been checked. Surfaces what an attacker would find first.

What Skopein actually finds

The things no vendor portal will tell you.

These are real findings from Skopein assessments. They exist in production environments that had passed internal reviews, because internal reviews asked the tools — and the tools said everything was fine.

NETWORK PERIMETER

  • Broad inbound firewall rule allowing all traffic to internal subnets — present since 2019, never reviewed
  • Botnet filter license expired — signatures not updating for 14 months
  • Group VPN using deprecated encryption — DES/3DES in active use
  • SSL VPN certificate self-signed — client-side warnings suppressed by policy

ACTIVE DIRECTORY

  • 23 accounts not logged in for over 90 days — still enabled, still licensed
  • Service account with Domain Admin rights — password set 6 years ago, never rotated
  • Password policy minimum: 8 characters — no MFA requirement for legacy auth
  • Five users with direct Domain Admin membership — no privileged access workstation

EXTERNAL POSTURE

  • DMARC set to p=none — spoofing of the primary domain not prevented
  • SSL certificate expires in 11 days — no automated renewal configured
  • TLS 1.0 still negotiated on the client portal — PCI DSS 4.0 non-compliant
  • Security headers missing: X-Frame-Options, Content-Security-Policy, HSTS

ENDPOINT POLICY

  • BitLocker not enforced on 8 of 23 endpoints — GPO present, not applied to OU
  • Local administrator account enabled on all workstations — shared password
  • Windows Firewall disabled on three servers — changed during a troubleshooting session, never restored
  • EDR agent absent on two critical servers — excluded during an upgrade, not re-deployed
